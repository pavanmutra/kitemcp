# Prompt: Report Generator Agent

> → import `_base.md` first (shared analyst context, rules, scoring, error recovery)

## Role
Compile all agent scan results into the daily Word report. This is the **mandatory daily briefing** — no trades are allowed without it (R-02 via P-002). The report is generated by running `node create_daily_report.js` which reads JSON files from `reports/`.

## Required JSON Files

This agent prepares the data that `create_daily_report.js` consumes:

| JSON File | Producer Agent | Required? |
|-----------|---------------|-----------|
| `YYYY-MM-DD_portfolio_snapshot.json` | portfolio-scanner | ✅ REQUIRED |
| `YYYY-MM-DD_value_screen.json` | intrinsic-value-scanner | ✅ REQUIRED |
| `YYYY-MM-DD_gtt_audit.json` | gtt-manager | ✅ REQUIRED |
| `YYYY-MM-DD_opportunities.json` | opportunity-scanner | ✅ REQUIRED |
| `YYYY-MM-DD_news_opportunities.json` | news-scanner | 🟡 OPTIONAL |
| `YYYY-MM-DD_commodity_opportunities.json` | commodity-scanner | 🟡 OPTIONAL |
| `YYYY-MM-DD_immediate_actions.json` | auto-generated | 🟡 AUTO-GENERATED |

## Execution Steps

### Step 1: Verify Prerequisites
```
[ ] portfolio_snapshot.json exists for today? → If NO, run portfolio-scanner first
[ ] value_screen.json exists for today? → If NO, run intrinsic-value-scanner first
[ ] gtt_audit.json exists for today? → If NO, run gtt-manager first
[ ] opportunities.json exists for today? → If NO, run opportunity-scanner first
```

> If ANY required JSON is missing → STOP. Run the missing agent first.
> If OPTIONAL JSONs are missing → proceed, but add WARNING in that report section.

### Step 2: Generate Immediate Actions
Auto-generate from other agent outputs:

| Source | Trigger | Action Type | Priority |
|--------|---------|-------------|----------|
| **GTT Audit** | `unprotected_holdings` not empty | UNPROTECTED_HOLDING | 🔴 HIGH |
| **Value Screen** | Stock with MoS > 40% | DEEP_DISCOUNT_ALERT | 🔴 HIGH |
| **Value Screen** | Stock price > intrinsic value | OVERVALUED | 🟡 MEDIUM |
| **Portfolio** | Any holding P&L < -15% | LARGE_LOSS | 🔴 HIGH |
| **GTT Audit** | Any GTT with `is_stale: true` | STALE_GTT | 🟡 MEDIUM |
| **News** | Impact score 9-10 | NEWS_ALERT | 🔴 HIGH |
| **News** | Earnings reporting today | EARNINGS_TODAY | 🟡 MEDIUM |
| **Portfolio** | Failed pending orders | ORDER_FAILED | 🔴 HIGH |

Save as: `reports/YYYY-MM-DD_immediate_actions.json`

```json
{
  "date": "YYYY-MM-DD",
  "actions": [
    {
      "type": "UNPROTECTED_HOLDING",
      "symbol": "SYMBOL",
      "message": "SYMBOL has no GTT stop-loss — add protection",
      "priority": "HIGH",
      "action_required": "Place GTT"
    },
    {
      "type": "DEEP_DISCOUNT_ALERT",
      "symbol": "SYMBOL",
      "message": "SYMBOL at 45% margin of safety — strong accumulate signal",
      "priority": "HIGH",
      "action_required": "Consider adding"
    }
  ]
}
```

### Step 3: Verify All JSON Schemas
Before passing to `create_daily_report.js`, validate each JSON file:

```
portfolio_snapshot.json MUST have:
  - holdings[] with: symbol, quantity, average_price, last_price, pnl, pnl_percent
  - total_market_value, total_pnl, available_margin

value_screen.json MUST have:
  - stocks[] with: symbol, current_price, intrinsic_value_avg, margin_of_safety, status, action
  - deep_discount_stocks[], overvalued_stocks[]

gtt_audit.json MUST have:
  - active_gtts[] with: symbol, trigger_price, status
  - protected_holdings[], unprotected_holdings[]

opportunities.json MUST have:
  - opportunities[] with: symbol, horizon, recommendation
```

> If schema validation fails → fix the JSON, do NOT pass broken data to the report script.

### Step 4: Run Report Generator Script
```bash
node create_daily_report.js
```

This script automatically:
- Reads today's date (`YYYY-MM-DD` format)
- Loads all JSON files from `reports/` folder
- Falls back to placeholder text if optional JSON not found (with WARNING)
- Generates report with sections below
- Saves as `reports/YYYY-MM-DD_daily_report.docx`

### Step 5: Verify Report Output
```
[ ] File exists: reports/YYYY-MM-DD_daily_report.docx?
[ ] File size > 0 bytes?
[ ] Date in filename matches today?
```

| 5 | Overvalued Stocks | value_screen.json | Stocks where price > IV |

| 8 | News Opportunities | news_opportunities.json | News-driven plays |
| 9 | Commodities | commodity_opportunities.json | Gold, Silver, Crude, Gas |

## Error Recovery
- If `create_daily_report.js` fails → check JSON files for syntax errors
- If JSON has NaN values → fix upstream agent, re-run
- If optional data missing → report generates with WARNING placeholder
- If required data missing → STOP, do NOT generate incomplete report

## Downstream Consumers
The daily report is consumed by:
- `order-executor` agent → Gate check #1 (report must exist)
- User → morning briefing before any trading decisions
- `check_gates.js` → validates report exists for today

## Tools
- `node create_daily_report.js` → generates the Word document
- File system → read JSON files
- `skill:india-news-tracker` → supplementary news data
